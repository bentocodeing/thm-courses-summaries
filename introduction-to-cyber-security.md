# Module 1 - Introduction to Cyber Security

# Defensive Security Intro

## What defensive security is

- Focuses on **protecting** an organisation’s systems, networks, and data from attacks.​
- Core activities: monitoring, detecting, responding, hardening systems, and learning from incidents.​


## Key areas of defensive security

- Monitoring and detecting: Continuous observation of network and system activity to spot suspicious behaviour (e.g., logins from unexpected countries).​
- Incident response: Coordinated process to contain and remove confirmed threats and restore normal operations.​
- Threat intelligence: Collecting information about attackers’ methods, targets, and trends to strengthen defences.​
- Vulnerability management: Finding and fixing software/system flaws, often with automated tools, to reduce attack surface.​
- Investigation and analysis: Distinguishing normal vs suspicious activity and digging into details like a puzzle to uncover what happened.​


## Main roles in a defensive team

- **SOC Analyst:** Monitors events on networks/systems, first line of defence, identifies suspicious or unexpected behaviour.​
- **Incident Responder:** Investigates and responds to active incidents, works in real time, and shares lessons learned to prevent future attacks.​
- **Security Engineer:** Builds and maintains tools and systems that enable monitoring, investigation, and exploration of security events.​
- **Digital Forensics:** Collects, preserves, and analyses evidence from systems and networks to understand what happened and how attackers operated.​

# Careers in Cybersecurity

## Security analysis roles (blue team)

- Security analyst: monitors alerts and logs to spot suspicious activity, then triages and escalates real threats.​
- Threat hunter: proactively searches networks and endpoints for hidden attackers using hypotheses and threat intel.​
- Incident responder: contains and eradicates attacks, then helps recover systems and write post-incident reports.​
- Common tools: SIEMs (e.g. Splunk), EDR/AV agents, log collectors, sandboxing and malware analysis tools.​


## Security engineering roles

- Security engineer: designs and builds secure networks, systems, and controls (firewalls, VPNs, identity, hardening).​
- Cloud security engineer: applies security principles to cloud platforms, focusing on identity, configuration, and monitoring.​
- DevSecOps engineer: integrates security checks into CI/CD pipelines and automates testing and deployment safeguards.​


## Penetration testing focus (offensive)

- Penetration tester: legally simulates attacks to find and report vulnerabilities before real attackers can exploit them.​
- Typical methodology: reconnaissance, scanning and enumeration, gaining initial access, privilege escalation, persistence, then reporting.​
- Core tools: port and service scanners, vulnerability scanners, exploit frameworks (e.g. Metasploit), password crackers, and web testing tools.​
- Web app testing often targets input validation, authentication, access control, and session handling weaknesses.​
- Red teamers emulate realistic adversaries over longer engagements, blending technical attacks with social engineering.​


## Skills and mindset for pen testers

- Strong networking, operating system, and scripting knowledge are essential foundations for offensive work.​
- Curiosity, persistence, and careful note-taking help track attack paths and turn findings into clear reports.​
- Understanding blue-team detection tools and processes improves stealthy attack simulation and higher-value findings.​
